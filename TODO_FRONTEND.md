**프론트엔드 리뉴얼 TODO**

- **목표**: 기존 프로젝트에서 `강의 데이터`와 `정원 감소 시뮬레이터`만 재활용하여, 프론트엔드만으로 동작하는 경량화된 수강신청 연습 페이지를 만든다.

- **우선순위 작업 목록**
  - **프로젝트 초기화 (필수)**: 새 폴더/파일 생성 (`index.html`, `js/app.js`, `css/styles.css`).
  - **데이터 통합**: 기존 `js/data_from_real.js`를 활용하거나 `data/lectures.json`로 변환하여 로드.
  - **강의 목록 UI**: 테이블 또는 카드 형태로 강의 표시. 필터(학부/학과/교과목명)와 검색 기능 제공.
  - **신청 흐름**: 신청 버튼 → 신청 성공/실패 토스트 또는 모달 → `localStorage`에 신청 목록 저장/조회.
  - **정원 감소 시뮬레이터**: 클라이언트에서 실행되는 시뮬레이터 구현(주기적 감소, 동시다발 감소 옵션, 감소량/속도 설정).
  - **트래픽 장애 시뮬레이터(UI)**: 장애 토글/모달, 실패율(%) 설정 UI — 장애 시 신청 요청을 강제로 실패시키도록 처리.
  - **UI/스타일링**: 간단하고 명료한 스타일(버튼, 테이블, 모달). 반응형 기본 적용.
  - **테스트 시나리오**: 다중 탭 동시 신청, 정원 0 처리, 장애 모드에서의 UX 확인.
  - **문서화**: 실행 방법(로컬에서 index.html 열기), 데이터 포맷 설명, 설정 가능한 시뮬레이션 옵션 정리.

- **데이터 모델(권장)**
  - id, dept, major, name, code, year, type, credit, cap(정원)

- **간단 실행 흐름**
  1. 브라우저에서 `index.html` 열기
  2. 강의 목록이 로드됨(정적 `data_from_real.js` 또는 JSON)
  3. 시뮬레이터 토글/설정 → 정원 감소 / 트래픽 장애 활성화
  4. 사용자는 신청 버튼을 눌러 신청을 시도(정원/장애 상태에 따라 성공 또는 실패)

- **권장 파일 구조**
```
project/
├─ index.html
├─ js/
│  ├─ app.js
│  ├─ data_from_real.js  # 기존 파일 재활용
│  └─ simulator.js      # 정원/트래픽 시뮬레이터
├─ css/
│  └─ styles.css
└─ README.md
```
